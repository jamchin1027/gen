CHPI Verilog Fix Review Ver.C

檔名：CHPI_verilog_fix_windows.html。這版以小面積為優先：寬 aligner/packer 只做組合暫存，DFF 只保留 residue；不用 q/n/w 當交付風格、parser 不用 for(k...)，並已重新跑 Icarus line testbench。
這次改什麼

正式 RTL 以 area-first readable hardware 為主，不再像軟體 trace code。

面積重點

raw DFF 26b→6b、RGB DFF 30b→23b、packet counter 9b→6b，debug path tie-off。

仿真已 PASS

5 byte CTRL_L + 960 byte RGB，一整行資料完整比對成功。

Ver.C 主流程 20b CHPI -> 24b RGB

CHPI[19:0] input

每 clock 20bit，LSB first，來源不提供 backpressure。

append aligner

DFF 只存 0..6bit raw_residue；本拍用組合 mux 臨時展開到 26bit。

parser stage0

若有 >=7bit，先取第一組 7bit；可能是 header，也可能是 payload。

parser stage1

接著取第二組 7bit，延續 stage0 後的 label/packet state。

parser stage2

最多再取第三組 7bit，守住 20/7 groups/clk 的連續吞吐。

restore label

payload 若為 {Label,0}/{Label,1}，還原成全 0 或全 1 的 7bit。

RGB24 packer

DFF 只存最多 23bit rgb_residue；30b pack_shift 只做組合暫存。

怎麼改 / 為什麼
這次重點：面積先砍

Ver.C 不再把 26b raw aligner 和 30b RGB packer 都做成暫存器。真正打 DFF 的只剩 raw_residue[5:0] 和 rgb_residue[22:0]，固定 flop 約從 107 bit 降到 75 bit。

variable shifter 改成明確 mux

chpi_in << raw_bit_count 改成 7-case placement；RGB payload 插入也改成 case-based function。合成器看到的是受限 mux，不是泛用 barrel shifter。

怎麼改 / 為什麼
去掉 q/n/w 交付風格

Rev.A 功能可跑，但你們團隊平常不一定用 *_q/*_n/*_w。Ver.C 延續這個修正，用有意義的訊號名，review 時比較像人在讀電路。

按功能拆 DFF

register 分成 raw residue、packet group counter、label state、RGB residue、RGB output，不再把寬暫存都丟進 DFF。

怎麼改 / 為什麼
for-loop 本身不是一定不能綜合

常數邊界 k<3 通常會被展開成硬體，但正式 review RTL 不保留這種寫法，避免被看成軟體流程。

三段 parser 明寫

20b/clk 需要平均 20/7 組 7bit，所以一拍最多吃 3 組。明寫 stage0/stage1/stage2 後，硬體吞吐與資料順序一眼就看得出來。

怎麼改 / 為什麼
避免 latch 疑慮

每個 combinational block 一開始先給完整 default，改到的 if 都補 else。這樣即使還是組合邏輯，也不是靠『保留上一值』在動。

組合邏輯變小塊

每一塊只負責 append、stage0、stage1、stage2、packer 其中一件事，review 時不用在一個巨大 always 裡追全局變數。

怎麼改 / 為什麼
小面積取捨

正式 RGB24 path 不需要 useful20 debug FIFO/counter。介面先保留，資料直接 tie-off，省掉一條旁路 datapath。

不讓 debug path 混淆主路徑

主路徑就是 CHPI[19:0] -> 7b parser -> restore -> RGB24 packer。debug output 不再看起來像核心電路的一部分。

修改前

  ...
  raw_fifo <= raw_fifo_next; // non-blocking

修改後 Ver.C
always @(posedge clk or negedge rst_n) begin
  if (!rst_n) begin
    rgb_valid <= 1'b0;
    rgb_data  <= 24'd0;
  end else begin
    rgb_valid <= pack_emit_valid;
    if (pack_emit_valid) begin
      rgb_data <= pack_emit_data;
    end else begin
      rgb_data <= 24'd0;

怎麼改 / 為什麼
clock block 只放 DFF

posedge block 裡只用 non-blocking 更新暫存器，不在同一個 block 裡用 blocking 算 next-state。

output 單一路徑

RGB output 由一個 DFF block 輸出，輸入只看 pack_emit_valid/pack_emit_data，不再多處寫 rgb_valid/rgb_data。

小面積修正重點 area-first RTL
DFF 只存 residue

raw 只存 0..6bit residue，RGB 只存 0..23bit residue；26b/30b 寬資料只在組合暫存中出現。

variable shift 拿掉

input align 改成 7-case placement，RGB payload 插入改成 case-based mux，避免泛用 barrel shifter。

group counter 取代 bit counter

420b packet 只需要數 header + 59 組 payload，因此用 6b group counter，不再用 9b bit position counter。

header 有扣掉

group count=0 的 7bit header 只更新 Label/H state，不送 RGB packer；後面 59 組才是 413b data。

    .chpi_in        (chpi_in),

module chpi_420b413b_rgb24_stream_dec (
  input  wire        clk,
  input  wire        rst_n,
  input  wire        valid_i,
  input  wire [19:0] chpi_in,
  output reg         rgb_valid,
  output reg  [23:0] rgb_data,
  output wire        useful20_valid,
  output wire [19:0] useful20_data
);
  // Synthesis-intended area-lean RTL.
  // CHPI[19:0] enters LSB first. Each 420b packet is:
  //   7b header {H, Label[5:0]} + 59 payload groups of 7b = 420b.
  // The header group updates label state and is not sent to the RGB packer.

  assign useful20_valid = 1'b0;
  assign useful20_data  = 20'd0;

  localparam [4:0] RAW_GROUP_BITS = 5'd7;
  localparam [4:0] CHPI_BUS_BITS  = 5'd20;
  localparam [4:0] RGB_WORD_BITS  = 5'd24;
  localparam [5:0] LAST_GROUP     = 6'd59;

  // Registered residue only. The wide 26b aligner exists only as
  // combinational temporary logic after appending the new 20b word.
  reg [5:0] raw_residue;
  reg [2:0] raw_bit_count;

  // 420b packet group counter:
  //   0      : header group
  //   1..59  : 59 decoded 7b payload groups
  reg [5:0] packet_group_count;
  reg [5:0] active_label;
  reg       restore_enable;

  // Registered RGB residue only. At clock edge this holds at most 23 bits.
  // The 30b packer is a combinational temporary for 23 old bits + 7 new bits.
  reg [22:0] rgb_residue;
  reg [4:0]  rgb_bit_count;

  reg [25:0] append_raw_shift;
  reg [4:0]  append_raw_count;
  reg [5:0]  raw_residue_next;
  reg [2:0]  raw_bit_count_next;

  wire        group0_valid;
  wire        group1_valid;
  wire        group2_valid;
  wire [6:0]  group0_bits;
  wire [6:0]  group1_bits;
  wire [6:0]  group2_bits;

  reg [5:0] st0_packet_group_count;
  reg [5:0] st0_label;
  reg       st0_restore_enable;
  reg       st0_payload_valid;
  reg [6:0] st0_payload_bits;

  reg [5:0] st1_packet_group_count;
  reg [5:0] st1_label;
  reg       st1_restore_enable;
  reg       st1_payload_valid;
  reg [6:0] st1_payload_bits;

  reg [5:0] st2_packet_group_count;
  reg [5:0] st2_label;
  reg       st2_restore_enable;
  reg       st2_payload_valid;
  reg [6:0] st2_payload_bits;

  reg [29:0] pack_shift;
  reg [4:0]  pack_count;
  reg [22:0] rgb_residue_next;
  reg [4:0]  rgb_bit_count_next;
  reg        pack_emit_valid;
  reg [23:0] pack_emit_data;

  assign group0_valid = (append_raw_count >= 5'd7);
  assign group1_valid = (append_raw_count >= 5'd14);
  assign group2_valid = (append_raw_count >= 5'd21);

  assign group0_bits = append_raw_shift[6:0];
  assign group1_bits = append_raw_shift[13:7];
  assign group2_bits = append_raw_shift[20:14];

  function [5:0] next_packet_group;
    input [5:0] group_now;
    begin
      if (group_now == LAST_GROUP) begin
        next_packet_group = 6'd0;
      end else begin
        next_packet_group = group_now + 6'd1;
      end
    end
  endfunction

  function [6:0] restore_payload_group;
    input [6:0] group_bits;
    input [5:0] label_bits;
    input       label_is_active;
    begin
      if (label_is_active && (group_bits[6:1] == label_bits)) begin
        if (group_bits[0]) begin
          restore_payload_group = 7'b1111111;
        end else begin
          restore_payload_group = 7'b0000000;
        end
      end else begin
        restore_payload_group = group_bits;
      end
    end
  endfunction

  function [29:0] insert_rgb_group;
    input [29:0] shift_in;
    input [4:0]  count_in;
    input [6:0]  group_bits;
    reg   [29:0] shift_tmp;
    begin
      shift_tmp = shift_in;
      case (count_in)
        5'd0:  shift_tmp[6:0]   = group_bits;
        5'd1:  shift_tmp[7:1]   = group_bits;
        5'd2:  shift_tmp[8:2]   = group_bits;
        5'd3:  shift_tmp[9:3]   = group_bits;
        5'd4:  shift_tmp[10:4]  = group_bits;
        5'd5:  shift_tmp[11:5]  = group_bits;
        5'd6:  shift_tmp[12:6]  = group_bits;
        5'd7:  shift_tmp[13:7]  = group_bits;
        5'd8:  shift_tmp[14:8]  = group_bits;
        5'd9:  shift_tmp[15:9]  = group_bits;
        5'd10: shift_tmp[16:10] = group_bits;
        5'd11: shift_tmp[17:11] = group_bits;
        5'd12: shift_tmp[18:12] = group_bits;
        5'd13: shift_tmp[19:13] = group_bits;
        5'd14: shift_tmp[20:14] = group_bits;
        5'd15: shift_tmp[21:15] = group_bits;
        5'd16: shift_tmp[22:16] = group_bits;
        5'd17: shift_tmp[23:17] = group_bits;
        5'd18: shift_tmp[24:18] = group_bits;
        5'd19: shift_tmp[25:19] = group_bits;
        5'd20: shift_tmp[26:20] = group_bits;
        5'd21: shift_tmp[27:21] = group_bits;
        5'd22: shift_tmp[28:22] = group_bits;
        5'd23: shift_tmp[29:23] = group_bits;
        default: shift_tmp = shift_in;
      endcase
      insert_rgb_group = shift_tmp;
    end
  endfunction

  // Append the new 20b word with a 7-case mux instead of a generic barrel
  // shifter. raw_bit_count is a registered residue and can only be 0..6.
  always @* begin
    append_raw_shift = {20'd0, raw_residue};
    append_raw_count = {2'd0, raw_bit_count};

    if (valid_i) begin
      append_raw_count = {2'd0, raw_bit_count} + CHPI_BUS_BITS;
      case (raw_bit_count)
        3'd0: append_raw_shift = {6'd0, chpi_in};
        3'd1: append_raw_shift = {5'd0, chpi_in, raw_residue[0]};
        3'd2: append_raw_shift = {4'd0, chpi_in, raw_residue[1:0]};
        3'd3: append_raw_shift = {3'd0, chpi_in, raw_residue[2:0]};
        3'd4: append_raw_shift = {2'd0, chpi_in, raw_residue[3:0]};
        3'd5: append_raw_shift = {1'd0, chpi_in, raw_residue[4:0]};
        3'd6: append_raw_shift = {chpi_in, raw_residue[5:0]};
        default: append_raw_shift = {6'd0, chpi_in};
      endcase
    end
  end

  // Keep only the post-parser residue in flops.
  always @* begin
    raw_residue_next   = append_raw_shift[5:0];
    raw_bit_count_next = append_raw_count[2:0];

    if (append_raw_count >= 5'd21) begin
      raw_residue_next   = {1'b0, append_raw_shift[25:21]};
      raw_bit_count_next = append_raw_count - 5'd21;
    end else if (append_raw_count >= 5'd14) begin
      raw_residue_next   = append_raw_shift[19:14];
      raw_bit_count_next = append_raw_count - 5'd14;
    end else if (append_raw_count >= 5'd7) begin
      raw_residue_next   = append_raw_shift[12:7];
      raw_bit_count_next = append_raw_count - 5'd7;
    end
  end

  // Stage 0: first possible 7b group in this clock.
  always @* begin
    st0_packet_group_count = packet_group_count;
    st0_label              = active_label;
    st0_restore_enable     = restore_enable;
    st0_payload_valid      = 1'b0;
    st0_payload_bits       = 7'd0;

    if (group0_valid) begin
      st0_packet_group_count = next_packet_group(packet_group_count);
      if (packet_group_count == 6'd0) begin
        st0_label          = group0_bits[5:0];
        st0_restore_enable = (group0_bits[5:0] != {6{~group0_bits[6]}});
        st0_payload_valid  = 1'b0;
        st0_payload_bits   = 7'd0;
      end else begin
        st0_label          = active_label;
        st0_restore_enable = restore_enable;
        st0_payload_valid  = 1'b1;
        st0_payload_bits   = restore_payload_group(group0_bits,
                                                   active_label,
                                                   restore_enable);
      end
    end
  end

  // Stage 1: second possible 7b group in this clock.
  always @* begin
    st1_packet_group_count = st0_packet_group_count;
    st1_label              = st0_label;
    st1_restore_enable     = st0_restore_enable;
    st1_payload_valid      = 1'b0;
    st1_payload_bits       = 7'd0;

    if (group1_valid) begin
      st1_packet_group_count = next_packet_group(st0_packet_group_count);
      if (st0_packet_group_count == 6'd0) begin
        st1_label          = group1_bits[5:0];
        st1_restore_enable = (group1_bits[5:0] != {6{~group1_bits[6]}});
        st1_payload_valid  = 1'b0;
        st1_payload_bits   = 7'd0;
      end else begin
        st1_label          = st0_label;
        st1_restore_enable = st0_restore_enable;
        st1_payload_valid  = 1'b1;
        st1_payload_bits   = restore_payload_group(group1_bits,
                                                   st0_label,
                                                   st0_restore_enable);
      end
    end
  end

  // Stage 2: third possible 7b group in this clock.
  always @* begin
    st2_packet_group_count = st1_packet_group_count;
    st2_label              = st1_label;
    st2_restore_enable     = st1_restore_enable;
    st2_payload_valid      = 1'b0;
    st2_payload_bits       = 7'd0;

    if (group2_valid) begin
      st2_packet_group_count = next_packet_group(st1_packet_group_count);
      if (st1_packet_group_count == 6'd0) begin
        st2_label          = group2_bits[5:0];
        st2_restore_enable = (group2_bits[5:0] != {6{~group2_bits[6]}});
        st2_payload_valid  = 1'b0;
        st2_payload_bits   = 7'd0;
      end else begin
        st2_label          = st1_label;
        st2_restore_enable = st1_restore_enable;
        st2_payload_valid  = 1'b1;
        st2_payload_bits   = restore_payload_group(group2_bits,
                                                   st1_label,
                                                   st1_restore_enable);
      end
    end
  end

  // RGB24 packer. The insert function is case-based, so synthesis sees a
  // bounded placement mux instead of a generic variable barrel shifter.
  always @* begin
    pack_shift         = {7'd0, rgb_residue};
    pack_count         = rgb_bit_count;
    pack_emit_valid    = 1'b0;
    pack_emit_data     = 24'd0;
    rgb_residue_next   = rgb_residue;
    rgb_bit_count_next = rgb_bit_count;

    if (st0_payload_valid) begin
      pack_shift = insert_rgb_group(pack_shift, pack_count, st0_payload_bits);
      pack_count = pack_count + RAW_GROUP_BITS;
      if (pack_count >= RGB_WORD_BITS) begin
        pack_emit_data  = pack_shift[23:0];
        pack_emit_valid = 1'b1;
        pack_shift      = pack_shift >> RGB_WORD_BITS;
        pack_count      = pack_count - RGB_WORD_BITS;
      end
    end

    if (st1_payload_valid) begin
      pack_shift = insert_rgb_group(pack_shift, pack_count, st1_payload_bits);
      pack_count = pack_count + RAW_GROUP_BITS;
      if (!pack_emit_valid && (pack_count >= RGB_WORD_BITS)) begin
        pack_emit_data  = pack_shift[23:0];
        pack_emit_valid = 1'b1;
        pack_shift      = pack_shift >> RGB_WORD_BITS;
        pack_count      = pack_count - RGB_WORD_BITS;
      end
    end

    if (st2_payload_valid) begin
      pack_shift = insert_rgb_group(pack_shift, pack_count, st2_payload_bits);
      pack_count = pack_count + RAW_GROUP_BITS;
      if (!pack_emit_valid && (pack_count >= RGB_WORD_BITS)) begin
        pack_emit_data  = pack_shift[23:0];
        pack_emit_valid = 1'b1;
        pack_shift      = pack_shift >> RGB_WORD_BITS;
        pack_count      = pack_count - RGB_WORD_BITS;
      end
    end

    rgb_residue_next   = pack_shift[22:0];
    rgb_bit_count_next = pack_count;
  end

  always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      raw_residue   <= 6'd0;
      raw_bit_count <= 3'd0;
    end else begin
      raw_residue   <= raw_residue_next;
      raw_bit_count <= raw_bit_count_next;
    end
  end

  always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      packet_group_count <= 6'd0;
    end else begin
      packet_group_count <= st2_packet_group_count;
    end
  end

  always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      active_label   <= 6'd0;
      restore_enable <= 1'b0;
    end else begin
      active_label   <= st2_label;
      restore_enable <= st2_restore_enable;
    end
  end

  always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      rgb_residue   <= 23'd0;
      rgb_bit_count <= 5'd0;
    end else begin
      rgb_residue   <= rgb_residue_next;
      rgb_bit_count <= rgb_bit_count_next;
    end
  end

  always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      rgb_valid <= 1'b0;
      rgb_data  <= 24'd0;
    end else begin
      rgb_valid <= pack_emit_valid;
      if (pack_emit_valid) begin
        rgb_data <= pack_emit_data;
      end else begin
        rgb_data <= 24'd0;
      end
    end
  end
endmodule

RTL Rule File for CHPI / DDIC Digital Helper Work
=================================================

Purpose
-------
This file records RTL mistakes found during the CHPI 413b/420b decoder review.
Before producing or revising RTL later, Codex should read this file first and
apply the checklist below.


Core Rules
----------

R001 - Do not present simulation-only RTL as production RTL.
  Mistake seen:
    Icarus PASS was described too close to "deliverable RTL".
  Required behavior:
    Every RTL output must be labeled as one of:
      - concept / trace code
      - simulation demo RTL
      - synthesis-intended RTL
      - production candidate RTL
  Verification requirement:
    If claiming synthesis-intended RTL, at minimum run syntax simulation and
    explicitly state that company lint/synthesis/timing sign-off is still
    required unless those tools were actually run.

R002 - Do not mix blocking and non-blocking assignment in one clocked process.
  Mistake seen:
    A posedge always block used blocking assignment for next-state variables
    and non-blocking assignment for flops in the same block.
  Required behavior:
    Clock/reset always blocks describe registers and must use non-blocking
    assignment for registered signals.
  Verification requirement:
    Search for "=" inside posedge/negedge state update blocks before delivery.
    Temporary combinational signals must not be assigned inside the DFF update
    block.

R003 - Match the team's readable RTL style; do not force q/n/w naming.
  Mistake seen:
    The first correction used a large next-state block with *_q, *_n, *_w
    naming. It was functionally clearer than the original, but it did not
    match this IC team's usual review style.
  Required behavior:
    Prefer meaningful signal names and split clock/reset always blocks by
    function or signal group:
      - raw input aligner
      - packet/header counter
      - label/restore state
      - RGB24 packer
      - RGB24 output
  Verification requirement:
    A reviewer should be able to draw the DFF groups directly from the always
    blocks without decoding a global q/n/w naming scheme.

R004 - Do not hide fixed datapath work inside a for-loop in final review RTL.
  Mistake seen:
    The area-lean parser used:
      for (k = 0; k < 3; k = k + 1)
    This is synthesizable when constant-bounded, but it reads like software and
    made the hardware structure less obvious.
  Required behavior:
    For this CHPI decoder, write the parser as explicit stage0 / stage1 /
    stage2 blocks. Each stage consumes at most one 7b group.
  Verification requirement:
    Final review RTL must not contain the parser loop or an unused k variable.

R005 - Avoid latch risk in combinational logic.
  Mistake seen:
    Earlier combinational style had many nested if branches and was hard to
    audit for complete assignment.
  Required behavior:
    Every combinational always block must assign all outputs at the top and
    every if branch must have an explicit else when it changes outputs.
  Verification requirement:
    Inspect each always @* block for complete defaults and explicit else paths.
    No combinational block may rely on an accidental retained value.

R006 - Make the hardware structure readable.
  Mistake seen:
    Input aligner, header/label FSM, restore datapath, and RGB packer were
    mixed together without clear hardware boundaries.
  Required behavior:
    The code must show the hardware stages in comments and naming:
      - input aligner
      - explicit 7b parser stage0 / stage1 / stage2
      - header/label state
      - restore datapath
      - RGB24 packer
      - optional debug observation path, if any
  Verification requirement:
    A digital engineer should be able to draw DFFs, muxes, counters, and
    datapath widths from the code without guessing.

R007 - Area is not simply "fewer operations per clock".
  Mistake seen:
    A one-group-per-clock decoder was considered, but continuous 20b input has
    no backpressure. That would need a large backlog buffer and may increase
    area.
  Required behavior:
    For fixed-rate input streams, preserve the required throughput first, then
    minimize buffers and remove non-production debug paths.
  Verification requirement:
    For CHPI[19:0], keep up with 20/7 groups per clock on average. The decoder
    therefore needs to consume up to three 7b groups per clock.

R008 - Avoid unnecessary debug datapath in production-intended RTL.
  Mistake seen:
    useful20 debug output had its own FIFO/counter datapath, increasing state
    and logic even though RGB24 is the real output.
  Required behavior:
    If a debug output is required by the interface but not production behavior,
    tie it off or isolate it behind an explicit parameter.
  Verification requirement:
    In the area-lean RTL, useful20_valid must be 1'b0 and useful20_data must be
    20'd0 unless the user explicitly asks to restore that debug path.

R009 - Keep buffer width justified by throughput and residue.
  Mistake seen:
    Buffer size was not explained clearly enough.
  Required behavior:
    State why each registered buffer and each temporary combinational bus
    exists.
  CHPI Ver.C area-lean decoder:
    raw_residue       : 6 registered bits, because after parsing up to three
                        7b groups the CHPI raw residue is only 0..6 bits.
    raw_bit_count     : 3 registered bits is enough for 0..6.
    append_raw_shift  : 26 combinational bits, because one clock may append
                        20b to 0..6 old residue bits.
    rgb_residue       : 23 registered bits, because RGB24 packer residue is
                        always less than 24 bits after optional output emit.
    rgb_bit_count     : 5 registered bits is enough for 0..23.
    pack_shift        : 30 combinational bits, because 23 old bits plus one
                        7b payload group can exist before emitting RGB24.

R010 - Header/label must be explicit state, not hidden temporary behavior.
  Mistake seen:
    label_reg and replace_en were blocking-assigned inside the clocked parser.
  Required behavior:
    The first 7b group of each 420b packet updates active_label and
    restore_enable. That header group must not enter the RGB data packer.
  Verification requirement:
    Packet counter reset point must be exact:
      count 0   : header {H, Label}
      count 7..413 : 59 payload groups
      count 413 payload group returns next count to 0

R011 - Output valid/data should have one clear generation path.
  Mistake seen:
    rgb_valid/rgb_data were assigned in several places inside the same clocked
    block.
  Required behavior:
    Generate RGB output from one output always block. If the internal packer is
    combinational, feed it with one clear pack_emit_valid / pack_emit_data
    result.
  Verification requirement:
    rgb_valid and rgb_data should be registered in one clock/reset block.

R012 - Use exact bit order and width comments.
  Mistake seen:
    Some explanations were unclear about LSB-first order and bus packing.
  Required behavior:
    For every encoder/decoder RTL:
      - state whether input is LSB-first or MSB-first
      - state input bus width
      - state output bus width
      - state packet/header widths
      - state residue/buffer strategy

R013 - Simulation must be run before updating user-facing HTML/site.
  Mistake seen:
    Documentation was polished before the RTL quality was sufficiently checked.
  Required behavior:
    For any HTML page that presents RTL as corrected, run the relevant testbench
    first and show:
      - simulator name
      - command
      - PASS/FAIL result
      - tested data scope

R014 - Minimize registered width before polishing RTL style.
  Mistake seen:
    A readable Ver.B implementation still kept the 26b raw aligner and 30b RGB
    packer as registered state. Functionally it passed, but it was not the
    smallest area style.
  Required behavior:
    For area-first stream RTL, only register the residue needed across clocks.
    Wide aligner/packer buses may exist as combinational temporary signals, but
    should not become flops unless timing closure requires pipelining.
  Verification requirement:
    Before delivering "area-lean" RTL, list the real flop state and check that:
      - CHPI raw residue is registered as 0..6 bits, not a full 26b aligner.
      - RGB residue is registered as 0..23 bits, not a full 30b packer.
      - Packet position uses the smallest natural counter; for CHPI 420b this
        is a 6b group counter for header + 59 payload groups.

R015 - Avoid generic variable shifters in small datapath glue.
  Mistake seen:
    Ver.B used variable shifts such as:
      chpi_in << raw_bit_count
      payload_bits << pack_count
    Synthesis may optimize them, but they can be read as barrel shifters and
    make area less predictable.
  Required behavior:
    If the shift amount has a small finite set, write explicit case placement
    or a case-based helper function.
  Verification requirement:
    Search final area-lean RTL for "<<" and ">>".
    Constant shifts are acceptable. Variable shifts in the main datapath need
    an explicit area reason.


Current Verified RTL
--------------------

File:
  chpi_long_decoder_rgb24_area_lean_verc.v

Purpose:
  Synthesis-intended, area-lean CHPI 420b to 413b decoder demo for continuous
  CHPI[19:0] input and RGB24 output.

Review-style choices in Ver.C:
  - Removed the parser for-loop and wrote explicit stage0/stage1/stage2 blocks.
  - Removed the *_q / *_n / *_w deliverable naming style.
  - Split registers into multiple clock/reset always blocks by function:
      raw residue, packet group counter, label state, RGB residue, RGB output.
  - Kept combinational helper blocks small and fully assigned.
  - Removed useful20 debug FIFO/counter datapath by tying debug output off.
  - Preserved continuous 20b throughput by parsing up to three 7b groups per
    clock.
  - Reduced real registered state:
      raw aligner DFF   : 26b -> 6b raw_residue
      raw count DFF     : 5b  -> 3b raw_bit_count
      packet counter DFF: 9b  -> 6b packet_group_count
      RGB packer DFF    : 30b -> 23b rgb_residue
  - Replaced generic variable input/RGB insertion shifts with case-based muxes.
  - Kept 26b raw aligner and 30b RGB packer only as combinational temporaries.
